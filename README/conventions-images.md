# Изображения

Все изображения для проекта должны храниться в директории `src/assets/images`. Важно поддерживать организованную структуру директорий для легкого поиска и управления изображениями.

- **Создание директорий**: Создавайте вложенные директории с осмысленными именами. Это помогает в быстром поиске и структурировании файлов.

- **Векторные изображения**: Векторные изображения делятся на иконки и прочие. Иконки должны храниться в директории `/src/assets/images/icons/`, где также возможно создание вложенных директорий. Остальные векторные изображения структурируйте аналогично растровым изображениям по секциям/компонентам/блокам.

## Форматы

Предпочтительный формат растровых изображений - `webp`. Другие форматы (например, `png`, `jpg`) считаются нежелательными. Использование `webp` обеспечивает лучшее качество при меньшем размере файла.

## Конвертация

Для конвертации и оптимизации изображений рекомендуется использовать сервис [Squoosh](https://squoosh.app). Этот инструмент поможет вам уменьшить размер изображений без потери качества.

## Оптимизация

- **Растровые изображения**: Если вы используете форматы растровых изображений, кроме `webp`, их оптимизация может быть выполнена с помощью плагина `vite-plugin-image-optimizer`, то есть отдельной оптимизацией таких изображений можно пренебречь.

- **GIF-файлы**: Оптимизацию файлов `gif` необходимо выполнять вручную либо получать оптимизированные файлы от дизайнера.

Следуя этим рекомендациям, вы обеспечите высокое качество и оптимизацию изображений в проекте.
