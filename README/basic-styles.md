# Базовые стили

Все файлы стилей импортируются в `/src/scss/style.scss`. Импорты сгруппированы по родительским директориям.

Ниже представлен краткий обзор каждого scss-файла присутствующего в сборке.

## Нормализация стилей

В сборке присутствует библиотека [`modern-normalize`](https://www.npmjs.com/package/modern-normalize). Фактически это то же что и [normalize.css](https://www.npmjs.com/package/normalize.css), но с радом преимуществ. Все преимущества описаны на странице пакета, но самое важное - данный пакет, в отличии от normalize.css, поддерживается.

Файл стилей из этой библиотеки подключен в `/src/js/main.js`. Это сделано специально, так как при подключении стилей в `/src/scss/style.scss` Vite перестаёт корректно отдавать source-map.

> 🤞 Если кто-то решит помочь с этой проблемой или знает решение – заведите issue и опишите ваши мысли на этот счёт.

## global

В директории `/src/scss/global/` присутствуют стили, которые, вероятнее всего не будут изменены на всём протяжении работы над проектом, разве, что вы захотите что-то добавить.

Это общие, не зависящие от проекта стили, которые задают основу простой и эффективной работы в дальнейшем.

Список файлов в базовой конфигурации в порядке из подключения:

- `./global/null` – обнуляющие стили.
- `./global/mixins` – миксины.
- `./global/functions` – функции.
- `./global/root` – стили корневого элемента.

## custom

Глобальные стили, именно под конкретный проект, представлены в директории `/src/scss/custom/`.

Описание файлов в порядке подключения:

- `./custom/variables` – здесь хранятся все переменные проекта, такие как отступы, радиусы, продолжительность анимаций. Переменные позволяют легко изменять значения в одном месте без необходимости менять их во всём коде. Так же переменные – это точка истины.
- `./custom/colors` – файл для определения цветовой палитры проекта. Рекомендуется хранить как базовые цвета, так и контекстные, например, $background-page, $accent. Это улучшает читаемость кода и упрощает поддержку цветовой схемы. Пример:

  ```SCSS
  $white: #fff;
  $green: #9acd32;

  $accent: $green;
  $background-page: $white;
  $background-accent: $accent;
  ```

- `./custom/fonts` — стили для подключения шрифтов через директиву @font-face. Здесь описываются подключаемые шрифты, их начертание, вес и форматы файлов.
- `./custom/typography` — стили типографики, определяющие базовые параметры текста, такие как шрифты для заголовков и основного текста, их размеры, межстрочные интервалы и отступы.
- `./custom/media` – в этом файле задана карта брейкпоинтов и миксин с помощью которого можно легко управлять стилями для различных экранов.
- `./custom/base` – базовые стили для страниц, например, такие как фоновый цвет для body, а также стили для контейнера.

## elements

Стили для UI-элементов, таких как кнопки, чек-боксы, инпуты и другие базовые компоненты, хранятся в директории `/src/scss/elements/`.

Каждый элемент имеет собственный файл и импортируется в `/src/scss/style.scss` в группу импортов, обозначенную как `// elements`.

## sections

В директории `/src/scss/sections/` находятся стили для секций — крупных независимых блоков страниц. Например, это могут быть стили для хедера, футера или секции с галереей.

## components

В `/src/scss/components/` хранятся стили для компонентов, таких как карточки, анонсы, хлебные крошки и прочие. Эти компоненты могут использоваться повторно в разных частях сайта.

Файлы импортируются в `/src/scss/style.scss` в группу `// components`.

## vendor

> ⚠️ Может быть удалены в следующих релизах. В будущем возможно решение, которое позволит исключить ручное изменение стилей библиотек. Или, что органично при работе с Vite, такие стили просто будут подключаться в js-файлы компонентов.

Директория `/src/scss/vendor/` используется для хранения переопределённых стилей сторонних библиотек.

Каждый файл импортируется в `/src/scss/style.scss` в группу импортов обозначенную комментарием `// vendor`.

## Порядок импорта из директорий elements и components

Правильный порядок подключения файлов важен, особенно если один элемент наследует стили другого.

Пример:

- сначала импортируется файл control, содержащий стили для текстового поля с лейблом и сообщением об ошибке.
- затем импортируется control-password, который расширяет стили control, добавляя свои специфические особенности для ввода пароля.
