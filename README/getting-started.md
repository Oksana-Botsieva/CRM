# Начало работы

## 1. Получение файлов репозитория

Рекомендуется начать работу с проектом выбрав один из предложенных вариантов

### Клонирование и настройка проекта

Для изучения работы сборки предпочтительнее использовать этот вариант, так как он быстрее, проще в реализации и вы сможете видеть историю коммитов.

1. Клонируйте репозиторий с помощью команды:

   ```bash
   git clone git@gitlab.com:relevant-team/sborka.git
   ```

2. Переименуйте директорию проекта в соответствии с названием нового проекта:

   ```bash
   mv sborka <название нового проекта>
   ```

3. Перейдите в директорию нового проекта:

   ```bash
   cd <название нового проекта>
   ```

4. Свяжите локальный репозиторий с новым удалённым репозиторием:

   ```bash
   git remote set-url origin <URL репозитория нового проекта>
   ```

### Инициализация проекта из архива

Этот вариант рекомендован, если необходимо избежать истории коммитов разработки самой сборки, они ни к чему в репозитории нового проекта.

1. Скачайте архив с файлами репозитория и распакуйте его. ![Скачать архив с файлами репозитория](/README/images/get-zip.png)

2. Перейдите в распакованную директорию.

3. Переименуйте директорию в соответствии с названием нового проекта:

   ```bash
   mv sborka <название нового проекта>
   ```

4. Инициализируйте новый репозиторий и привяжите его к удалённому репозиторию:

   ```bash
   git init
   git remote add origin <URL репозитория нового проекта>
   ```

## 2. Очистка проекта

После установки зависимостей с помощью `npm install`, запуска dev-сервера и проверки корректности работы сборки, перед началом работы над новым проектом, следует очистить проект от demo. Сделать это можно, или вручную, или запустив скрипт очистки.

Если вы недавно начали работать со сборкой, рекомендуется хотя бы несколько раз выполнить очистку проекта вручную для лучшего понимания структуры данных и зависимостей переменных.

### Очистка с помощью скрипта

> ⚠️ Если вы читаете этот текст, это значит, что на текущий момент скрипт еще не разработан. Очистите проект вручную.

### Очистка вручную

> 💡 При очистке проекта вручную не забывайте добавлять файлы .gitkeep. [Зачем?](https://wiki.merionet.ru/articles/cto-takoe-gitkeep-kak-otslezivat-i-dobavliat-pustye-papki-v-git)

Список действий для очистки проекта вручную:

- остановите dev-сервер если он запущен с помощью комбинации клавиш `⌃ Control` + `C` (или `ctrl` + `C` на Windows) в терминале.
- удалите файлы шрифтов из `/src/assets/fonts/`.
- удалите файлы изображений из `/src/assets/images/`. Директорию `/src/assets/images/icons/` удалять не стоит.
- удалите файл `/src/components/demo.hbs`.
- очистите тег `<main>` в `/src/index.html`.
- удалите строку `<link rel="icon" type="image/svg+xml" href="./assets/images/vite.svg" />` в `/src/index.html`.
- удалите файл `/src/js/utils/counter.js`.
- удалите импорт и запуск функции `setupCounter` из `/src/js/main.js`.
- удалите файл `/src/scss/components/_demo.scss`.
- очистите файл `/src/scss/custom/_base.scss`.
- очистите файл `/src/scss/custom/_colors.scss` или настройте переменные цветов под ваш проект.
- очистите файл `/src/scss/custom/_fonts.scss` или настройте шрифты под ваш проект.
  > 💡 Не забудьте о том, что в сборке предусмотрен миксин `font-face` для для подключения шрифта.
- измените `$breakpoints` в файле `/src/scss/custom/_media.scss` под ваш проект или оставьте как есть.
- очистите файл `/src/scss/custom/_typography.scss` или настройте базовую типографику под ваш проект.
- очистите файл `/src/scss/custom/_variables.scss` или настройте переменные под ваш проект.
  > ⚠️ Важно! Переменные `$transition-func` и `$transition-duration` используются как значения по умолчанию в миксине `transition`.
