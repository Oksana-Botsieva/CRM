# Sborka

**Sborka** - это мощная сборка для разработки фронтенд-части веб-приложений, предназначенная для дальнейшей интеграции в CMS системы, такие как 1C-Битрикс. Этот проект предоставляет все необходимые инструменты для создания высококачественных и легко интегрируемых веб-приложений с минимальными усилиями.

Сборка включает в себя:

- **Интеграция с Vite**: Быстрая и удобная сборка проекта с использованием Vite, что обеспечивает быстрое время сборки и разработку с поддержкой горячей перезагрузки.
- **Настройка ESLint, Prettier и Stylelint**: Инструменты для обеспечения высокого качества кода и соблюдения код-стилей.
- **Шаблонизация с Handlebars**: Поддержка шаблонизатора Handlebars для создания динамических HTML-шаблонов, что упрощает разработку и повторное использование компонентов.
- **Поддержка SCSS**: Гибкая настройка стилей с использованием SCSS для создания модульных и поддерживаемых стилей.
- **Оптимизация изображений**: Оптимизация изображений с помощью `vite-plugin-image-optimizer`, что улучшает качество и производительность без конвертации форматов.

Эта сборка создана для облегчения процесса разработки фронтенда и обеспечения интеграции с CMS системами. Используйте её для быстрого старта проектов и следуйте лучшим практикам и стандартам в разработке.

## Начало работы

Рекомендуется начать работу с проектом выбрав один из предложенных вариантов

### Клонирование и настройка проекта

Для изучения работы сборки предпочтительнее использовать этот вариант, так как он быстрее, проще в реализации и вы сможете видеть историю коммитов.

1. Клонируйте репозиторий с помощью команды:

   ```bash
   git clone git@gitlab.com:relevant-team/sborka.git
   ```

2. Переименуйте директорию проекта в соответствии с названием нового проекта:

   ```bash
   mv sborka <название нового проекта>
   ```

3. Перейдите в директорию нового проекта:

   ```bash
   cd <название нового проекта>
   ```

4. Свяжите локальный репозиторий с новым удалённым репозиторием:

   ```bash
   git remote set-url origin <URL репозитория нового проекта>
   ```

### Инициализация проекта из архива

Этот вариант рекомендован, если необходимо избежать истории коммитов разработки самой сборки, они ни к чему в репозитории нового проекта.

1. Скачайте архив с файлами репозитория и распакуйте его. ![Скачать архив с файлами репозитория](./README/Images/get-zip.png)

2. Перейдите в распакованную директорию.

3. Переименуйте директорию в соответствии с названием нового проекта:

   ```bash
   mv sborka <название нового проекта>
   ```

4. Инициализируйте новый репозиторий и привяжите его к удалённому репозиторию:

   ```bash
   git init
   git remote add origin <URL репозитория нового проекта>
   ```

## Версионность

- **Node.js**: v22.8.0

## Скрипты

Сборка предоставляет следующие скрипты для работы с проектом:

- `prepare`: Настройка Husky для автоматического запуска хуков Git.
- `lint`: Запуск линтеров ESLint, Prettier и Stylelint для проверки и исправления кода.
- `dev`: Запуск дев-сервера Vite для разработки.
- `build`: Выполнение линтинга и сборка проекта с помощью Vite.
- `preview`: Предварительный просмотр сборки с помощью Vite.
- `deploy:demo`: Сборка проекта и деплой на демо-сервер с помощью `deploy.js`.
- `deploy:prod`: Сборка проекта и деплой на продакшн-сервер с помощью `deploy.js`.

## Работа с npm

- Установка пакетов: `npm install`
- Запуск dev-сервера: `npm run dev`
- Сборка билда для продакшна: `npm run build`
- Запуск статичного сервера на основе финального бандла в директории `dist`: `npm run preview`
- Деплой (demo-сервер): `npm run deploy:prod`
- Деплой (production-сервер): `npm run deploy:prod`

## Соглашения

Эти соглашения помогают поддерживать порядок и единообразие в проекте. Следуя этим рекомендациям, мы обеспечиваем легкость в управлении проектом и поддержку высокого качества кода.

### Изображения

Все изображения для проекта должны храниться в директории `src/assets/images`. Важно поддерживать организованную структуру директорий для легкого поиска и управления изображениями.

- **Создание директорий**: Создавайте вложенные директории с осмысленными именами. Это помогает в быстром поиске и структурировании файлов.

- **Векторные изображения**: Векторные изображения делятся на иконки и прочие. Иконки должны храниться в директории `/src/assets/images/icons/`, где также возможно создание вложенных директорий. Остальные векторные изображения структурируйте аналогично растровым изображениям по секциям/компонентам/блокам.

#### Форматы

Предпочтительный формат растровых изображений - `webp`. Другие форматы (например, `png`, `jpg`) считаются нежелательными. Использование `webp` обеспечивает лучшее качество при меньшем размере файла.

#### Конвертация

Для конвертации и оптимизации изображений рекомендуется использовать сервис [Squoosh](https://squoosh.app). Этот инструмент поможет вам уменьшить размер изображений без потери качества.

#### Оптимизация

- **Растровые изображения**: Если вы используете форматы растровых изображений, кроме `webp`, их оптимизация может быть выполнена с помощью плагина `vite-plugin-image-optimizer`, то есть отдельной оптимизацией таких изображений можно пренебречь.

- **GIF-файлы**: Оптимизацию файлов `gif` необходимо выполнять вручную либо получать оптимизированные файлы от дизайнера.

Следуя этим рекомендациям, вы обеспечите высокое качество и оптимизацию изображений в проекте.
